# macOS 権限設定ガイド

JXA からマウス操作や他アプリケーションの制御を行うためには、macOS のプライバシー設定で権限を付与する必要があります。

## 1. アクセシビリティ

マウスカーソルの移動・クリック、キーボードイベントの送信を行うスクリプトに必要です。

1. `システム設定` を開く
2. `プライバシーとセキュリティ` → `アクセシビリティ` を選択
3. 右下の `+` ボタンから、スクリプトを実行する以下のアプリを追加して有効化する
   - `ターミナル` (`/System/Applications/Utilities/Terminal.app`)
   - もしくは `iTerm` などお使いのターミナル
   - VS Code から実行する場合は `Visual Studio Code`

## 2. オートメーション

他のアプリケーション（Finder、Safari、Chrome 等）を JXA で操作する場合に必要です。

1. `システム設定` → `プライバシーとセキュリティ` → `オートメーション`
2. ターミナルアプリの下に、操作対象アプリ（例: `System Events`、`Finder`）が表示されるので、それぞれにチェックを入れる

## 3. 入力監視（任意）

カーソル座標やキー入力をリアルタイムで監視するスクリプトを使う場合に必要です。

1. `システム設定` → `プライバシーとセキュリティ` → `入力監視`
2. ターミナルアプリを追加して有効化

## トラブルシューティング

- 権限を有効にした直後はターミナルを再起動するか、Mac を再起動すると確実に反映される
- スクリプト実行時に `osascript: execution error: Not authorized to send Apple events to ...` のようなエラーが出る場合は、上記「オートメーション」の設定漏れを確認する
- macOS のメジャーアップデート後は、権限がリセットされることがあるため再度設定する
